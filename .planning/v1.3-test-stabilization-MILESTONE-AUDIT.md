## REQ coverage

### TESTFIX-01 — `homes-areas.spec.ts` area rename race ✓

Un-skipped. Root cause: `page.goto('/areas')` firing before the
Server Action POST from Save committed → re-query missed the
in-flight DB write. Fix: `page.waitForResponse()` gate + bumped
final `toBeVisible` timeout 5s → 10s.

### TESTFIX-02 — `notifications.spec.ts` conditional reveal race ✓

Un-skipped. Root cause: RHF `Controller` checkbox → `useWatch` →
conditional render of dependent field, committed on subsequent
React 19 tick. Fix: assert `toBeChecked({ timeout: 5_000 })` first,
then `toBeVisible({ timeout: 10_000 })` on the revealed child.

### TESTFIX-03 — `createServerClientWithRefresh` retry ✓

**The big one.** What the v1.2-security audit called "10 pre-
existing failures" was actually 18 assertions across 12 specs, all
inheriting the same race: under CI's rate-limit-disabled
concurrency, PB's `authRefresh` endpoint intermittently rejects a
just-issued token. `authStore.clear()` → `isValid === false` →
layout redirects to `/login` → test expecting `/h` times out.

Fix: single retry with 150ms pause in the `authRefresh` catch arm
of `lib/pocketbase-server.ts`. Transient PB errors resolve within
~50-100ms in practice, so one retry absorbs them. Happy path is
untouched; the retry only fires on the error edge. Hits every
trust-boundary entry point (signup, login, session restore) in
one place vs. per-test band-aids.

### TESTFIX-04 — Shared E2E helpers ✓

Added three flake-resistant primitives to `tests/e2e/helpers.ts`:

- **`signup(page, email, password, name?)`** — canonical signup
  with `waitForURL(/\/h(\/|$)/, { timeout: 15_000 })`. Absorbs the
  Phase 36 retry window.
- **`waitForServerAction(page, { trigger, urlPattern?, timeout? })`** —
  generic click + POST-response-wait + URL-settle wrapper.
- **`waitForRHFConditional(triggerLocator, revealLocator, opts?)`** —
  RHF conditional-reveal wrapper with trigger-state-first +
  reveal-with-timeout.

**Not force-migrated** across the 10 spec files that still have
inline `signup()` copies — those work because Phase 36 fixed the
underlying race. Migration is opportunistic; Phase 37 SUMMARY
documents the rationale.

### TESTFIX-05 — CI flake-retry budget ✓

**`playwright.config.ts`:** `retries: process.env.CI ? 2 : 0` +
JSON reporter on CI.

**`.github/workflows/ci.yml`:** new "Flake budget check" step
parses `test-results/e2e-report.json` and fails the build if any
test required a retry to pass. Outcome: retry mechanism catches
genuine transient blips (so random infra noise doesn't page
anyone), but masquerading-as-stable is impossible because the
retry is always surfaced.

**On this milestone's closing commit:** zero retries consumed. No
E2E test needed the retry. The budget check is insurance, not a
crutch.

## Lessons worth keeping

1. **Source-of-truth fixes beat per-test band-aids.** The triage
   agent offered three fix options for TESTFIX-03. Option A
   (server-side retry) + Option C (test-level waits) both work;
   Option A ALSO fixes 17 other tests that weren't mentioned in
   the triage, and future tests inherit the fix for free. Don't
   hunt the symptom; fix the race.

2. **Batching flake fixes is legitimate when dependencies chain.**
   Phase 35's un-skipped tests exercise `signup()` → would regress
   immediately without Phase 36. Shipping them as one commit
   saved a CI cycle and avoided a "predictable fail on the way
   to green" noise.

3. **The flake-budget check is cheap and powerful.** Parsing
   Playwright's JSON report + failing CI if any retry was consumed
   is ~40 lines of shell. That small gate prevents slow-drift
   flake accumulation — the thing that turned into "10 pre-
   existing failures" by the time anyone noticed.

4. **Opportunistic migration is fine for low-value mechanical
   edits.** 10 inline `signup()` copies are effectively equivalent
   post-fix. Forcing a migration commit for cleanliness is
   churn-for-appearance. The lesson isn't "skip migration" — it's
   "migrate when you're in the file anyway for another reason;
   don't open 10 files just for style."
