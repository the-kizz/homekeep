# v1.3 — Test Stabilization Roadmap

**Theme:** green the E2E suite, zero skipped tests, zero flaky
retries. No new features, no dep bumps beyond what's needed for
fixes.

| Phase | Focus | REQ-IDs | Estimated effort |

## Phase execution order rationale

**35 first (trivial wins)** — fastest path to un-skip two tests;
unblocks other spec files that currently run with neighbors
effectively sandboxed around the skip.

**36 next (the real bug)** — root-causes and fixes the auth race,
which is the single highest-impact item. 18 downstream assertions
stop being flaky. Soak for 2 CI runs before moving on.

**37 (abstraction)** — only AFTER 36 is confirmed stable. Build
helpers FROM the working fix, not before, so the abstraction
matches what actually works.

**38 (guardrails + cleanup)** — flake-retry budget prevents
regression into v1.4. Planning dir tidy is orthogonal but natural to
bundle since it's pure hygiene.
