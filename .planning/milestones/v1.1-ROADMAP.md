## Key Accomplishments

1. **`computeNextDue` 6-branch composition (LOAD-15 hard gate, 21/21 green).** archived → override → smoothed → seasonal-dormant → seasonal-wakeup → OOFT → natural cycle/anchored. All pairwise interactions explicitly tested.
2. **`placeNextDue` forward-only smoother** with tolerance window `min(0.15 × freq, 5)`, deterministic tiebreakers (lowest-load → closest-to-ideal → earliest), <100ms budget for 100-task household (observed 3.78ms, 26× headroom).
3. **Atomic write semantics preserved end-to-end.** `completeTaskAction` batches {completion + override consume + OOFT archive + smoothed placement} in a single `pb.createBatch()` — all four ops roll back together on any failure.
4. **Action-sheet reschedule UX.** `<RescheduleActionSheet>` + `<ExtendWindowDialog>` ship the "Just this time" / "From now on" model. `reschedule_marker` field drives Phase 17 REBAL preservation.
5. **SPEC v0.4 + AGPL drift fix.** License corrected MIT → AGPL-3.0-or-later; full v1.1 changelog documenting every new field, collection, algorithm, and UI surface. Release-ready documentation.
6. **Disposable-PB integration coverage.** Port allocation 18090..18105, zero collisions, 16 live-PB integration suites covering migration + atomic write + end-to-end flows.

## Carried Forward to v1.1.1 (Patch Phase)

Three interacting bugs documented in audit — single patch phase required:

1. **PB 0.37.1 NumberField 0-vs-null quirk** for `active_from_month` / `active_to_month` — dashboards map via `?? null` which doesn't catch `0`.
2. **Year-round seasonal-wakeup** — fresh task with window `1..12` returns Jan 1 next year instead of natural cadence.
3. **`placeNextDue` self-counting** — target task appears in its own load map during placement; breaks idempotency when bug #1 is fixed in isolation.

All three surfaced via Phase 16 visual UAT. Not release-blockers for rc1; scheduled for rc2 / v1.1.1 patch phase.
