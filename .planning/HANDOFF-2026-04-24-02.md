## What the smoke test found (real bugs)

### Bug #1: Auth cookie dropped over HTTP

- **Symptom:** Signup lands on `/h`, but clicking "Create your first home" bounces to `/login?next=/h/new`. Session invisible.
- **Root:** `lib/actions/auth.ts` set `secure: process.env.NODE_ENV === 'production'`. On the VPS (`SITE_URL=http://46.62.151.57:3000`, `NODE_ENV=production`), `Secure=true` → browsers drop the cookie over plain HTTP.
- **Fix:** Secure derived from `SITE_URL.startsWith('https://')`. Added `isSecureSite()` helper. Commit `3215ca8`.
- **Bonus:** Remember Me checkbox on login form. Unchecked → session cookie (no maxAge). Default checked → 14-day persistent. Wired through `loginAction`.

### Bug #2: Next.js `/api/*` routes 404 behind Caddy

- **Symptom:** `/api/csp-report` returns 404 via port 3000 (Caddy); returns 204 when hit directly on port 3001 (Next.js).
- **Root:** `docker/Caddyfile` had a catch-all `handle /api/* { reverse_proxy localhost:8090 }` (PocketBase). Next.js API routes (csp-report, admin scheduler, demo session) fell through to PB and 404'd.
- **Fix:** Explicit allow-list BEFORE the PB catch-all:
  ```
  handle /api/health      { reverse_proxy localhost:3001 }
  handle /api/csp-report  { reverse_proxy localhost:3001 }
  handle /api/admin/*     { reverse_proxy localhost:3001 }
  handle /api/demo/*      { reverse_proxy localhost:3001 }
  handle /api/*           { reverse_proxy localhost:8090 }  ← PB catch-all
  ```
- Commit `731acc5`. Live-verified: `curl -sI -X POST http://127.0.0.1:3000/api/csp-report` → `204`.
