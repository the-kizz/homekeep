<objective>
Invite + member server actions + public /invite/[token] route + signup-next flow + member-gated mutation swap. This Wave delivers every HOME-05/HOME-06/HOME-07 server-side behavior behind an end-to-end integration test that proves the owner → invite → second-user-signup → accept → shared-home roundtrip on a live disposable PocketBase.

Purpose: With Wave 1's DB-layer gate in place (every mutation is guarded by the membership rule via `_via_` back-relation), the server actions become thin preflight + discriminated-union result wrappers. The acceptInvite atomicity guarantee (`pb.createBatch` + UNIQUE INDEX on `home_members (home_id, user_id)` + UNIQUE INDEX on `invites.token`) collapses four separate race-condition surfaces (Pitfall 4 double-accept, Pitfall 1 orphan membership, Pitfall 5 signup-next loss, Pitfall 6 post-removal stale assignee) into a single DB transaction. The signup-next thread-through turns `/invite/TOKEN → /signup → /invite/TOKEN → /h/ID` into a boring, linearizable redirect chain.

Output: 4 new server action files + 2 schema files + 3 lib utilities + 1 public route + `.env.example` updates + 4 existing action files modified + 4 unit/integration tests.

    ```typescript
    const TOKEN_TTL_MS = 30 * 60_000; // 30 minutes
    let cached: { pb: PocketBase; expiresAt: number } | null = null;

    export async function createAdminClient(): Promise<PocketBase> {
      if (cached && Date.now() < cached.expiresAt && cached.pb.authStore.isValid) {
        return cached.pb;
      }
      const email = process.env.PB_ADMIN_EMAIL;
      const password = process.env.PB_ADMIN_PASSWORD;
      if (!email || !password) {
        throw new Error('PB_ADMIN_EMAIL / PB_ADMIN_PASSWORD not configured');

      const pb = new PocketBase('http://127.0.0.1:8090');
      await pb.collection('_superusers').authWithPassword(email, password);
      cached = { pb, expiresAt: Date.now() + TOKEN_TTL_MS };
      return pb;
    }

    export function resetAdminClientCache(): void { cached = null; }
    ```

|----------|-------------|
| Invitee browser → `/invite/[token]` → `acceptInvite` server action | Token travels in URL path (public). Rate-limited at PB's `/api/` label; token entropy is 192-bit so brute-force is infeasible. |
