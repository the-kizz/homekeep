<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → static manifest/icons/offline.html | Public assets served without auth; must not leak anything new |
| browser → service-worker scope "/" | SW intercepts every same-origin request post-install; a misconfigured runtime cache could serve stale auth pages |
| client ↔ localStorage (dismissed_insecure_banner) | UI-state only; not an auth signal — must stay non-security-bearing |
| HTTP deploy → user | Browser denies isSecureContext, denies PWA install, denies SW registration — app must degrade, not break |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01-01 | Information Disclosure | public/manifest.webmanifest | accept | Manifest contains only name/display/colors/icon paths — no tokens, no PII. Public by design (every PWA manifest is). |
| T-07-01-02 | Tampering | app/sw.ts runtime cache of authenticated pages | mitigate | Serwist `defaultCache` uses NetworkFirst for navigations, so the SW cannot pin a stale logged-in page after logout. Do NOT widen precache to `/h/**` (would persist user-state across accounts); rely on NetworkFirst + navigationPreload. |
